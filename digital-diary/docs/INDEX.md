# Documentation Index

A map of the project's documentation.

| Document                     | Purpose                                                            |
| ----------------------------- | ------------------------------------------------------------------- |
| `README.md`                   | Project overview, stack, and setup instructions                    |
| `AGENTS.md`                   | Guidance for AI agents continuing development                      |
| `docs/constitution.md`        | Non-negotiable product and design principles                       |
| `docs/design-system.md`       | Typography, color, spacing, and motion specification                |
| `docs/definition-of-done.md`  | Exact scope and acceptance criteria for the current milestone       |
| `docs/roadmap.md`             | Multi-phase implementation roadmap                                  |
| `docs/adr.md`                 | Architecture Decision Records log                                   |

## Suggested Reading Order

1. `README.md` — get the project running.
2. `docs/constitution.md` — understand what this product is and isn't.
3. `docs/design-system.md` — understand the visual rules before touching UI.
4. `docs/definition-of-done.md` — understand what is and isn't in scope right
   now.
5. `AGENTS.md` — practical conventions for making changes.
